## 编译器版本查看

### 查看默认支持的C/C++标准版本

在Linux环境下，可以使用以下命令查看编译器默认支持的标准版本：

```bash
# 查看 GCC 支持的 C 标准版本
$ gcc -dM -E -x c /dev/null | grep -F STDC_VERSION
#define __STDC_VERSION__ 201710L

# 查看 G++ 支持的 C++ 标准版本  
$ g++ -dM -E -x c++ /dev/null | grep -F __cplusplus
#define __cplusplus 201703L
```

**版本对照表：**

| C 标准 | `__STDC_VERSION__` 值 |
|--------|----------------------|
| C90    | 未定义               |
| C99    | 199901L              |
| C11    | 201112L              |
| C17    | 201710L              |

| C++ 标准 | `__cplusplus` 值 |
|----------|------------------|
| C++98    | 199711L          |
| C++11    | 201103L          |
| C++14    | 201402L          |
| C++17    | 201703L          |
| C++20    | 202002L          |

## 符号类型详解

### 强符号和弱符号

在C语言中，无论是变量名还是函数名，在编译器看来都只是一个符号。编译器根据符号的特性将其分为两类：

#### 强符号（Strong Symbol）
- **函数名**：所有已定义的函数
- **初始化的全局变量**：声明时就赋予了初值的全局变量

```c
// 强符号示例
int initialized_var = 42;        // 强符号：初始化的全局变量
void my_function() {              // 强符号：函数定义
    // 函数体
}
```

#### 弱符号（Weak Symbol）  
- **未初始化的全局变量**：声明但未赋初值的全局变量

```c
// 弱符号示例
int uninitialized_var;           // 弱符号：未初始化的全局变量
extern int external_var;         // 弱符号：外部声明
```

### 符号强弱的本质

**强符号之所以"强"**：
- 拥有确切的数据和定义
- 变量有明确的初值
- 函数有具体的函数体实现

**弱符号之所以"弱"**：
- 尚未被完全定义或初始化
- 没有确切的数据内容
- 在链接时可以被强符号覆盖

## 链接规则与冲突解决

### 符号冲突的三种场景

在多文件工程项目中，对于相同的全局变量名或函数名，可能出现以下三种情况：

1. **强符号 + 强符号**
2. **强符号 + 弱符号** 
3. **弱符号 + 弱符号**

### 链接器处理规则

链接器遵循以下三个基本规则来解决符号冲突：

#### 1. 一山不容二虎
**规则**：不能同时存在两个强符号

**示例**：
```c
// 文件1: main.c
int global_var = 10;              // 强符号
void func() { /* ... */ }         // 强符号

// 文件2: utils.c  
int global_var = 20;              // 强符号 - 链接错误！
void func() { /* ... */ }         // 强符号 - 链接错误！
```

**结果**：链接器报告重定义错误，编译失败。

#### 2. 强弱可以共处
**规则**：强符号和弱符号可以同时存在，选择强符号

**示例**：
```c
// 文件1: main.c
int global_var = 100;             // 强符号

// 文件2: utils.c
int global_var;                   // 弱符号
```

**结果**：链接器选择强符号，`global_var` 的值为 100。

#### 3. 体积大者胜出  
**规则**：都是弱符号时，选择占用内存空间更大的符号

**示例**：
```c
// 文件1: main.c
int global_array[10];             // 弱符号，40字节

// 文件2: utils.c  
int global_array[100];            // 弱符号，400字节
```

**结果**：链接器选择较大的符号，分配400字节空间。

### 实践建议

#### 避免符号冲突的最佳实践

1. **使用static关键字**：将不需要外部访问的符号声明为静态
```c
static int internal_var = 0;     // 文件内部符号，不会冲突
```

2. **规范的头文件声明**：在头文件中只放声明，在源文件中放定义
```c
// header.h
extern int global_var;           // 声明

// source.c
int global_var = 42;             // 定义
```

3. **使用命名空间或前缀**：避免全局符号名称冲突
```c
int module_a_counter = 0;
int module_b_counter = 0;
```

### 调试符号冲突

#### 使用nm命令查看符号
```bash
# 查看目标文件中的符号
$ nm object_file.o

# 符号类型说明：
# T - 强符号（代码段）
# D - 强符号（数据段）  
# B - 弱符号（BSS段）
# U - 未定义符号
```

#### 使用objdump查看详细信息
```bash
# 查看符号表
$ objdump -t object_file.o

# 查看重定位信息
$ objdump -r object_file.o
```

## 总结

理解强符号和弱符号的概念对于C语言程序开发至关重要，特别是在处理大型多文件项目时。正确运用这些概念可以：

- **避免链接错误**：理解符号冲突的产生原因
- **优化项目结构**：合理组织代码避免不必要的全局符号
- **调试链接问题**：快速定位和解决符号相关的编译问题
- **提高代码质量**：遵循最佳实践编写更健壮的代码

通过掌握编译器版本查看、符号类型识别和链接规则，可以更好地进行C语言项目开发和维护。
