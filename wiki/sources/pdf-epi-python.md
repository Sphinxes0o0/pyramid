# Elements of Programming Interviews in Python (Sampler)

> This PDF is a sampler/book excerpt of the EPI Python book. Despite being placed in the C++ slides directory, it is **not a C++ conference slide** — it is a Python programming interview book. It is ingested here as a general algorithms/data-structures resource.

## Content Overview

The full EPI Python book covers **three major parts:**

### Part I: Data Structures and Algorithms (Chapters 1–16)

| Chapter | Topic | Example Problem |
|---------|-------|-----------------|
| 1 | Primitive Types | Computing the parity of a word |
| 2 | Arrays | Dutch national flag problem |
| 3 | Strings | Interconvert strings and integers; Base conversion |
| 4 | Linked Lists | Test for cyclicity |
| 5 | Stacks and Queues | Stack with max API; Binary tree level-order |
| 6 | Binary Trees | Height-balanced tree test |
| 7 | Heaps | Merge sorted files |
| 8 | Searching | First occurrence of k in sorted array |
| 9 | Hash Tables | Anonymous letter constructibility |
| 10 | Sorting | Array intersection; Calendar rendering |
| 11 | Binary Search Trees | BST property verification |
| 12 | Recursion | Power set generation |
| 13 | Dynamic Programming | 2D array traversal counting |
| 14 | Greedy Algorithms | 3-sum problem |
| 15 | Graphs | Boolean matrix painting (BFS/DFS) |
| 16 | Parallel Computing | Timer class implementation |

### Part II: Domain-Specific Problems (Chapters 17–20)

- **Design Problems** — e.g., copyright infringement detection system, design patterns
- **Language Questions** — closures, shallow vs deep copy
- **Object-Oriented Design** — creational patterns (Singleton, Factory, Builder, Prototype)
- **Common Tools** — version control merging

### Part III: The Honors Class (Chapter 21)

Advanced problems: GCD computation, maximum product of all entries but one, etc.

## Key Approach

EPI Python follows the same structured methodology as the original EPI (C++):
- Each problem has a **problem statement**, **hints**, and **solution analysis**
- Solutions cover **brute-force → optimized** progression
- All code is in **Python 3**
- A companion website (elementsofprogramminginterviews.com) includes known errors and updates
