# 进程与线程

## 定义

进程是分配资源的基本单位（内存、文件），线程是轻量级进程，仅分配 CPU 执行资源，由操作系统直接调度。

## 关键要点

- **资源分配**: 进程拥有完整地址空间、文件资源；线程共享进程资源（堆栈除外），仅分配 PC/寄存器/栈
- **创建开销**: 创建进程需分配完整内存空间、分段（堆/栈/正文）；创建线程只需分配栈和寄存器
- **内核级 vs 用户级**: Linux PThread 是用户级线程（库实现）；KThread 是内核级线程（直接调度）
- **状态机**: 就绪(Ready) → 运行(Running) → 阻塞(Block)；就绪无法直接变阻塞，运行无法直接变阻塞
- **分时调度**: 时间片轮转，`CPU 核心数 << 进程/线程数`
- **上下文切换**: 中断信号 → 保存寄存器 → 执行调度 → 恢复寄存器 → 切换执行
- **进程表 vs 线程表**: 进程表记录内存映射、文件、地址空间；线程表记录状态、优先级、PC、寄存器
- **多核并行**: 并行(parallel) 要求多核同时执行；并发(concurrent) 一段时间内交替执行

## 进程 vs 线程开销

| 方面 | 进程 | 线程 |
|------|------|------|
| 内存分配 | 完整地址空间，多段 | 仅栈+寄存器 |
| 创建成本 | 高（大量初始化） | 低（轻量） |
| 切换开销 | 大（切换地址空间） | 小（共享地址空间） |
| 通信 | 需 IPC 机制 | 直接共享内存 |
