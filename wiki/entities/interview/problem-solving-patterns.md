# 问题解决模式 (Problem Solving Patterns)

## 定义

编码面试中的通用解题思路和模式，帮助在面试中快速识别问题类型并选择正确的解决方法。

## 解题四步法

### Step 1: 理解问题
- 输入是什么？类型、范围、约束
- 输出是什么？期望的格式和类型
- 特殊要求？时间限制、空间限制
- 边界情况？空输入、大规模输入、重复元素

### Step 2: 暴力解法
- 先想出一个能工作的解法
- 评估其时间和空间复杂度
- 这是 baseline，后续会优化

### Step 3: 优化
- **空间换时间** — 使用额外数据结构
- **时间换空间** — 预处理、预计算
- **算法改进** — 分治、动态规划、贪心
- **数据结构改进** — 用哈希表、用堆优化

### Step 4: 代码实现
- 保持代码清晰和模块化
- 使用有意义的变量名
- 适当注释关键逻辑

## 常见问题模式

### 两指针模式
适用于有序数组或链表问题。
- 典型场景：两数之和、三数之和、验证回文串

### 滑动窗口
适用于子数组/子串问题。
- 典型场景：最大子数组长度、字符串排列、最小覆盖子串

### 二分查找变体
适用于有序数据的搜索问题。
- 典型场景：搜索旋转排序数组、寻找边界、寻找峰值

### 递归与回溯
适用于组合、排列、子集问题。
- 典型场景：全排列、组合总和、N皇后问题

### BFS (广度优先搜索)
适用于层次遍历、最短路径问题。
- 典型场景：二叉树层序遍历、图的最短路径

### DFS (深度优先搜索)
适用于遍历、连通性、路径问题。
- 典型场景：二叉树遍历、图的连通分量、岛屿数量

### 动态规划
适用于最优子结构问题。
- 典型场景：斐波那契、背包问题、最长公共子序列

### 分治法
将问题分解为子问题，解决后合并。
- 典型场景：归并排序、快速排序、最近点对

## 复杂度速查

| 数据结构 | 操作 | 复杂度 |
|---------|------|--------|
| 数组 | 访问 | O(1) |
| 数组 | 查找 | O(n) |
| 链表 | 插入/删除(头部) | O(1) |
| 哈希表 | 查找/插入/删除 | O(1) 平均 |
| 二叉搜索树 | 查找/插入/删除 | O(log n) 平均 |
| 堆 | 查找最大/最小 | O(1) |
| 堆 | 插入/删除 | O(log n) |

## 面试技巧

1. **不要急于编码** — 先思考清楚再动手
2. **边想边说** — 展示你的思维过程
3. **寻求确认** — "这样理解对吗？"
4. **考虑边界** — 空输入、极端情况
5. **多种方案** — 先说简单方案，再优化
