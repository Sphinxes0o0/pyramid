# 结构型设计模式 (Structural Patterns)

## 定义

结构型模式关注**类和对象的组合**，通过继承和组合来形成更大的结构。

## 关键模式

### 适配器模式 (Adapter)

**定义**：将一个类的接口转换成客户期望的另一个接口。

**核心**：接口兼容，解决不同 API 接口的兼容性问题
**特点**：适配器像转接头，不改变原有功能

### 桥接模式 (Bridge)

**定义**：将抽象部分与实现部分分离，使它们都可以独立变化。

**核心**：抽象与抽象之间的分离（区别于继承的纵向扩展）
**本质**：实体和行为分离，满足 DIP 原则

**适用场景**：
- 平台独立性（JDBC 驱动）
- 跨平台兼容性
- 运行时切换不同实现

### 组合模式 (Composite)

**定义**：用树形结构处理对象之间的复杂关系。

**核心**：部分-整体关系，单个对象和组合对象一致对待
**适用场景**：文件系统、GUI 组件、Organization 结构

### 装饰模式 (Decorator)

**定义**：动态地向现有对象添加新功能，不改变其结构。

**核心**：包装器模式，在外层扩展
**特点**：链式结构，装饰器不超过 10 个

**对比**：
- vs 适配器：适配器侧重转换，装饰器侧重扩展
- vs 桥接：桥接横向宽度扩展，装饰纵向深度扩展

**适用场景**：
- API 安全性扩展（加解密、压缩）
- final 类的功能扩展

### 门面模式 (Facade)

**定义**：为子系统中的一组接口提供统一的高层接口。

**核心**：API 网关，提供统一入口
**适用场景**：高可用性网关、简化复杂系统调用

### 享元模式 (Flyweight)

**定义**：通过共享对象减少内存加载消耗。

**核心**：细粒度对象的共享
**适用场景**：大量相似对象的场景（文本编辑器字符、游戏中子弹）

### 代理模式 (Proxy)

**定义**：控制和管理对象的访问。

**类型**：远程代理、虚代理、保护代理、智能引用
**适用场景**：延迟加载、访问控制、日志记录

## 结构型模式对比

| 模式 | 核心思想 | 扩展方向 |
|------|---------|---------|
| 适配器 | 接口转换 | 兼容旧接口 |
| 桥接 | 抽象分离 | 多维度独立演化 |
| 组合 | 树形结构 | 部分-整体 |
| 装饰 | 包装扩展 | 纵向深度 |
| 门面 | 统一入口 | 简化调用 |
| 享元 | 对象共享 | 节约内存 |
| 代理 | 访问控制 | 间接访问 |
